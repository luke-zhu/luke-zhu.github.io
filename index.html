Seeing the Big Picture with Software Architecture

We all love it when that one person in a meeting once again tells everybody to “take a step back and think about the big picture.” I honestly would be that person for my team projects, but I’m currently stuck playing the role of “socially-incompetent programmer.” As cliche the advice to “think big picture” is, there is a lot of merit to it. Software projects generally fail not because the source code is terrible, but because of higher-level issues in the solution design and the management of the project.

Coding More Quickly

Write test cases. Identify cognitive load, then simplify the problem. Identify the quality level your code should reach. Do things more carefully. Get used to your tools Don’t learn new things. Some ways to code more quickly: Write a test case to verify that your code works. Why: If you’re like me, you celebrate when your code work on the first try. If your code is marginally complicated, It’s less costly to make sure your code works before it’s in production.

Avoiding data science

I’ve been a teaching assistant for a Data Science Master’s program for the the past 3 semesters. However, I have absolutely zero plans to try to become a data scientist. Here are some of the reasons why: Nearly all job positions require you to have an advanced degree even though common daily duties (A/B testing, training ML models, etc.) require of what’s taught in school to do well. Some say that the theory doesn’t matter but they want someone who can get the work done without hand holding.
